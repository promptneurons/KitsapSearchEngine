# DocumentClassifier Skill

**Purpose:** Classify incoming files by type for ETL routing and GLN assignment
**Version:** 0.3

---

## Overview

Three-stage classification:
1. **Format classification** — detect file type by extension/content for ETL routing
2. **GLN classification** — predict the Generalized Luhmann Number for organizational filing
3. **GLN relevance ranking** — rank documents by proximity in GLN metric space

## Stage 1: Format Classification Matrix

| Extension | MIME | Inferred Type | Pipeline |
|-----------|------|---------------|----------|
| `.csv` | text/csv | financial, tabular | parse-csv → JSONL |
| `.qif`, `.qfx`, `.ofx` | application/x-* | financial | financial-import |
| `.xml` | text/xml | gnucash, config | schema-detect |
| `.json`, `.jsonl` | application/json | data | validate → store |
| `.pdf` | application/pdf | document | extract-text → archive |
| `.md`, `.txt` | text/* | document | store |
| `.zip`, `.tar.gz` | application/* | archive | extract → recurse |
| `.png`, `.jpg` | image/* | media | metadata → archive |

## Stage 2: GLN Classification (Triple-Axis Resolver)

**Script:** `scripts/gln-resolver.py`

Predicts the closest GLN for any document using three scoring axes:

| Axis | Weight | Signal | Data Source |
|------|--------|--------|-------------|
| Archetype | 50% | Filename prefix (SPEC-, INC-, MQL-, etc.) + path components | PREFIX_TO_GLNS + DIR_TO_GLNS lookup tables |
| FGID | 30% | Content keywords → department codes (LD, MG, SF, MK, LG, CF, FN) | FGID_KEYWORDS keyword lists |
| CEC | 20% | Content → Cutter Expansive Classification subject class | `data/cec-hierarchy.ttl` + `data/jdn-ontology.ttl` crosswalk |

### Usage

```bash
# Classify a file
python3 scripts/gln-resolver.py projects/BUSINESS-PLAN.md

# Classify by title + content (no file needed)
python3 scripts/gln-resolver.py --title "MQL-015-evolanzoltan" \
    --content "lead scoring for substack author"
```

### Output

Rich YAML with prediction, axis breakdown, and ranked candidates:

```yaml
prediction:

  name: leads
  path: clawd/projects/casefiles/leads
  confidence: 0.74

  archetype:
    prefix: MQL-
    matched_subtrees: [a1c9g5, a1c9g5d]
    score: 1.0
  fgid:
    detected: [MG, LG]
    node_fgid: [N100, LD, MG, LN]
    score: 0.33
  cec:
    class: B
    label: "Philosophy, Paedeia, Logic, Religion"
    jdn: 27
    score: 0.5
candidates:
  - gln: a1c9g5d
    name: leads
    score: 0.74
  - gln: a1c9g5
    name: casefiles
    score: 0.73
```

### Data Dependencies

- `llms-N200-index.yaml` — the GLN tree (loaded at startup)
- `data/cec-hierarchy.ttl` — CEC subject classes (parsed with regex)
- `data/jdn-ontology.ttl` — JDN ↔ CEC crosswalk (parsed with regex)

No network calls. Requires only stdlib + pyyaml.

## Stage 3: GLN Relevance Ranking

### Precompute

**Script:** `scripts/gln-precompute.py`

Builds `data/gln-cache.jsonl` from all 803 indexed files by running each through the GLN resolver and extracting a snippet (first non-heading paragraph, ~200 chars).

```bash
# Build the cache (runs ~803 files, takes ~30s)
python3 scripts/gln-precompute.py

# Limit for testing
python3 scripts/gln-precompute.py --limit 50
```

### Ranker

**Script:** `scripts/gln-ranker.py`

Ranks documents by proximity in a 4-axis metric space derived from precomputed GLN resolver vectors:

| Axis | Weight | Function |
|------|--------|----------|
| GLN proximity | 40% | Shared prefix length / max GLN length |
| FGID Jaccard | 25% | Department code set overlap |
| CEC match | 20% | 1.0 same class, 0.5 same JDN, 0.3 same letter, 0.0 otherwise |
| Candidate overlap | 15% | Jaccard of top-5 candidate GLN sets |

### Usage

```bash
# Random mode: pick 11 docs, rank 10 against the first
python3 scripts/gln-ranker.py

# Specific query document
python3 scripts/gln-ranker.py --query projects/BUSINESS-PLAN.md

# Reproducible selection
python3 scripts/gln-ranker.py --seed 42

# Custom pool size
python3 scripts/gln-ranker.py --pool 20
```

### Output

Markdown to stdout with search-engine-style snippets, score breakdown tables, and visual relevance bars.

### Data Dependencies

- `data/gln-cache.jsonl` — precomputed vectors (generated by `gln-precompute.py`)
- Requires cache to have at least `pool + 1` entries

No network calls. Requires only stdlib + json.

## Integration with ETL

```
File arrives
    ↓
Stage 1: Format classify (extension + content inspection)
    ↓
Route to appropriate pipeline (parse-csv, financial-import, etc.)
    ↓
Stage 2: GLN classify (scripts/gln-resolver.py)
    ↓
File to organizational location by GLN
    ↓
Stage 3: Precompute vectors (scripts/gln-precompute.py)
    ↓
Relevance ranking available (scripts/gln-ranker.py)
    ↓
Load to S3 / Virtuoso
```
